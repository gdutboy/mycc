# SOP for AI 写作指南

## 核心原则

1. **读者是 AI**：SOP 是写给 AI 执行的，不是给人读的，必须精确可解析
2. **AI 做所有能做的**：能自动化就自动化，必要时才打扰用户
3. **关键节点透传**：服务中断、凭证信息、超时兜底、完成通知必须告知用户
4. **引用而非重复**：已有文档直接引用，不复制粘贴

## 执行流程

收到 SOP 编写请求后：

1. 确认操作目标和执行者（哪个 AI、哪个场景）
2. 按模板结构起草 SOP（前置检查 → 步骤 → 完成后 → 故障排查）
3. 用写作检查清单自检（见 REFERENCE.md）
4. 输出文件，前缀 `[sop4ai]`

## SOP 结构模板

```markdown
# [操作名称] SOP

> 这是给 AI 看的操作流程，AI 自己执行，用户只需要确认。

## 前置检查
[版本检查、环境检查等]

## Step 1: [步骤名]
### 1.1 执行动作
\`\`\`bash
# AI 执行的命令
\`\`\`
### 1.2 告知用户（如需要）
[需要用户知道的信息]

## 完成后
- [ ] 通知用户（飞书/终端）
- [ ] 验证结果

## 故障排查
[常见问题和解决方案]
```

## 边界

- 单篇 SOP 编写不超过 20 分钟
- 不写传统人读 SOP，不省略故障排查段落
