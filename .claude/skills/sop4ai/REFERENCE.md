# SOP for AI — 参考文档

## 核心理念详解

### 读者是 AI，不是人

SOP 是写给**用户的 AI** 看的，不是给用户看的。

| 传统 SOP | SOP for AI |
|----------|------------|
| 告诉人怎么操作 | 告诉 AI 怎么替人操作 |
| 人读懂就行 | AI 能解析执行 |
| 可以模糊 | 必须精确 |

### AI 做所有能做的

```
AI 能做的 → 直接做
需要用户决策的 → 问用户
需要用户知道的 → 告诉用户
```

### 关键节点透传

AI 执行过程中，这些信息必须告诉用户：

| 信息类型 | 示例 |
|----------|------|
| **会中断的服务** | "后端重启，约 2-5 分钟不可用" |
| **用户可能需要的凭证** | "配对码：xxx，连接码：xxx" |
| **超时兜底** | "5 分钟没恢复请回电脑前检查" |
| **完成通知** | 用飞书/邮件通知用户 |

### 引用而非重复

```markdown
# 好
**请先阅读并执行**：[FAQ 第 1 条](../docs/FAQ.md)

# 不好
（把 FAQ 内容复制过来）
```

---

## 写作检查清单

写完 SOP 后，逐项检查：

- [ ] **AI 能独立执行吗？** — 命令完整、路径明确
- [ ] **用户需要知道什么？** — 中断时间、凭证、兜底方案
- [ ] **有引用已有文档吗？** — 不要重复造轮子
- [ ] **完成后有通知吗？** — 别让用户干等
- [ ] **有故障排查吗？** — 出问题 AI 能自己解决

## 不做的事

- 不写给人看的传统 SOP（那不是本 skill 的职责）
- 不在 SOP 中重复已有文档的内容（用引用）
- 不省略故障排查段落
