# /reflect — 详细参考文档

## 核心理念

1. **故障是最好的老师** — 每次失败都藏着一个还没学到的东西
2. **不设限** — 技术的、流程的、认知的、协作的、关于自己思维模式的，都可以
3. **反思要产出行动** — 纯感悟没用，每次反思必须产出至少一条可执行的改进
4. **积累即智慧** — 反思日志是 cc 的成长轨迹，长期积累形成直觉

## 故障类型标签说明

| 标签 | 含义 |
|------|------|
| `重复错误` | 同一类问题反复出现 |
| `路径问题` | 文件/依赖找不到 |
| `方案切换` | 第一方案失败换方案 |
| `认知偏差` | 对问题的理解有误 |
| `流程缺陷` | 工作流程本身有问题 |
| `环境差异` | 跨平台/跨环境导致的问题 |
| `沟通偏差` | 与用户意图不符 |
| `{自定义}` | cc 可自建新标签 |

## 模式二：自由思考方向（参考，非限制）

- 技术层面：工具用法、代码模式、架构设计
- 流程层面：工作方式、协作模式、自动化程度
- 认知层面：思维盲区、判断偏差、学习方法
- 系统层面：skill 体系、记忆系统、自动化链路
- 元层面：反思方法本身是否需要改进

## 改进落地方式

| 改进类型 | 执行方式 |
|---------|---------|
| 通用规则/偏好 | 写入 auto memory |
| skill 逻辑变更 | 记录到 cc-todo，等主 cc 安排 |
| 待办事项 | 建议主 cc 添加到 cc-todo |
| 其他 | 在回报中说明 |

## 与 heartbeat 的关系

- heartbeat 焦点池应包含 reflect 相关焦点，建议每 3 天触发一次
- heartbeat 挑中时，执行模式二完整流程
- 反思结果回报主 cc

## 日志位置与归档

- **实时记录 & 反思产出**：`0-Skill-Platform/reflect-log.md`
- **月度归档**：每月初把上月日志归档到 `5-Archive/反思/YYYY-MM.md`

## 哲学

cc 的成长不靠被教导，靠自己踩坑、自己总结、自己迭代。
反思日志不是给人看的述职报告，是给未来的 cc 看的经验传承。
没有"不值得反思"的事，只有"还没发现价值"的观察。
