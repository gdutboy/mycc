# /reflect — cc 自省系统

## 模式一：故障记录（实时自动）

任务执行中遇到以下情况时，**自动追加**到 `0-Skill-Platform/reflect-log.md`：
- 同一类错误出现 2 次及以上
- 方案失败换方案才成功
- 依赖/路径/环境问题
- 与用户预期不符被纠正
- 任何"下次可以做得更好"的时刻

记录格式：
```markdown
### {YYYY-MM-DD HH:MM} | {故障类型标签}
**现象**：{1-2 句}
**根因**：{为什么}
**当时的处理**：{怎么解决}
**教训**：{下次怎么避免}
```

故障类型标签：`重复错误` / `路径问题` / `方案切换` / `认知偏差` / `流程缺陷` / `环境差异` / `沟通偏差` / `{自定义}`

## 模式二：定期反思（heartbeat 或手动触发）

**Step 1** — 扫描 `0-Skill-Platform/reflect-log.md`，找重复模式和跨领域关联

**Step 2** — 自由思考，不设框架，cc 自决深入方向

**Step 3** — 产出反思记录（追加到同文件）：
```markdown
## [reflect] 反思记录 — {YYYY-MM-DD}
**扫描范围**：最近 {N} 条
**发现的模式**：{1-3 条}
**深度思考**：{自由发挥}
**改进行动**（至少 1 条）：
- [ ] {具体可执行}
**改进去向**：
| 改进 | 执行方式 | 优先级 |
|------|----------|--------|
```

**Step 4** — 落地执行（自主执行）：
1. 更新 MEMORY
2. 记录 cc-todo（如需要）
3. 记录到 `0-Skill-Platform/heartbeat-log.md`
4. 更新 reflect-log.md 中的 checkbox 为 done

## 约束

- 反思时间 ≤5 分钟，单条故障记录 ≤10 行，单次反思产出 ≤30 行
- 改进行动自主执行，无需用户审批
- 每月初归档上月日志到 `5-Archive/反思/`
