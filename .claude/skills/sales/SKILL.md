# 顶级销售 DCFCS 框架

## 第一步：采集输入

从用户输入中提取（不够就问，最多问一轮）：
- **必须**：客户画像（行业/角色/规模）、客户痛点、我方产品/服务
- **可选**：已有案例、当前阶段（默认首次接触）、约束（预算/竞品）

## 第二步：判断阶段，确定输出重点

| 阶段 | 重点步骤 |
|------|---------|
| 首次接触 | D（诊断）+ F（钩子）为主 |
| 已接触 1-3 次 | C（确定性）+ C（跟进计划）为主 |
| 即将成交 | C（确定性）→ 临门一脚话术 |
| 已成交 | S（裂变）为主 |
| 不确定 | 全流程均衡输出 |

## 第三步：按 DCFCS 生成策略

- **D - Diagnose 诊断**：3-5 个探测性问题 + 1 句冲击力诊断句（带量化数字）
- **C - Certainty 确定性**：案例话术（同行成功故事）+ 风险消除方案 + ROI 测算
- **F - Free Hook 免费钩子**：设计 1-2 个低成本高感知的钩子 + 话术
- **C - Chase System 系统跟进**：按时间线规划 5+ 次接触，每次带新价值，附跟进话术模板
- **S - Spread 裂变**：超预期交付策略 + 转介绍时机和话术 + 证言采集方式

## 第四步：输出产出

按以下格式输出，产出带 `[sales]` 前缀，长度 ≤ 100 行：

```
## [sales] 销售策略：[客户名/场景]
### 客户画像（行业/角色/痛点/阶段）
### D - 诊断策略（探测问题 + 冲击力诊断句）
### C - 确定性武器（案例话术 + 风险消除 + ROI）
### F - 免费钩子（方案 + 话术）
### C - 跟进计划（时间线表格）
### S - 裂变设计（超预期策略 + 转介绍话术）
### 注意事项
```

## 边界

- 执行时间 ≤ 5 分钟
- 重大商务决策（合同/报价）升级为 L3 通知
