# 极简产品设计法

> 苹果设计哲学的本质：用户不需要理解系统，系统要理解用户。

## 边界

- 资源预算：≤10 分钟；审视一个产品/功能
- 产出长度：≤50 行
- 不做：不画原型图；不写代码；不做完整 PRD

---

## 执行流程

当用户描述一个产品/功能设计时，按以下步骤审视：

1. **找到"一件事"**：用户打开这个产品，最想知道/做的一件事是什么？
2. **列出所有操作**：把当前设计里所有需要用户手动操作的地方列出来。
3. **逐个审问**：每个操作——能自动做吗？用户真的需要手动触发吗？使用频率高/中/低？
4. **合并同类项**：把做类似事情的按钮合并成一个。
5. **极端场景下沉**：把 <5% 用户才需要的功能用小字链接藏到底部。

---

## 产出格式

所有产出以 `[fun-design]` 前缀标注。

```markdown
## [fun-design] 极简审视报告

**用户关心的一件事**：[xxx]

**当前按钮/操作清单**：
| 按钮/操作 | 能否自动 | 使用频率 | 建议 |
|----------|---------|---------|------|
| xxx | 能/不能 | 高/中/低 | 保留/自动化/下沉/删除 |

**简化方案**：
- Before: [描述当前状态，几个按钮]
- After: [描述简化后状态]

**检验清单**：
- [ ] 最常见路径需要点几下？（目标：0-1 下）
- [ ] 有没有按钮可以用自动化替代？
- [ ] 有没有两个按钮在做类似的事？
- [ ] 有没有 <5% 场景的功能占据了显眼位置？
- [ ] 用户需要理解任何技术概念吗？
```
