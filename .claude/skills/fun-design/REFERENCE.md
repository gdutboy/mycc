# fun-design 参考文档

## 核心原则

### 1. 用户只关心一件事
找到用户真正关心的那**一件事**，其他都是噪音。

### 2. 能自动就不要手动
每一个按钮都是设计的失败。问：这个操作能不能自动完成？

### 3. 状态自动判断
系统应该知道当前是什么状态，自动切换行为。用户不需要选择。

### 4. 按钮合并：5 → 1
多个按钮做类似的事 → 合并成一个。

### 5. 极端场景用小字链接
99% 用户不需要的功能，用底部小字藏起来。

---

## 典型案例

| 产品 | 用户关心的 | 用户不关心的 |
|------|------------|--------------|
| mycc | 能用还是不能用 | 连接码、配对码、token |
| 相机 | 拍出好照片 | ISO、光圈、快门速度 |
| 打车 | 车来了没 | 调度算法、司机接单逻辑 |

**按钮简化案例**：
- "重试连接" 按钮 → 自动每 5 秒检测
- "刷新" 按钮 → 实时更新
- "保存" 按钮 → 自动保存

## 不做的事

- 不画原型图或 UI 设计稿
- 不写代码实现
- 不做完整 PRD
- 不做用户调研（那是 /user-journey 的事）
