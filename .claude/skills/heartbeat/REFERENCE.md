# heartbeat — 详细参考

## 饥饿值计算说明

```
hunger = wait_hours × frequency_weight
```

- `wait_hours`：当前时间距"上次"列的小时数。"-" 表示从未执行，用创建日期算。
- `frequency_weight`：见 heartbeat-pool.md 底部频率参考表

多个焦点饥饿值相近（差距 <10%）时，优先挑产出去向是 A区的焦点。

## cc-todo ready 条件判断

| 条件模式 | 判断方式 |
|----------|---------|
| "每周X" | 今天星期几？ |
| "每月X号" | 今天几号？ |
| "{ID} done" | 该 ID 在 cc-todo 里的状态 |
| "文件存在" / "产出文件存在" | Glob 检查路径 |
| 括号里有锚点提示 | 按提示验证 |

## 回报格式

### f12（cc-todo 扫描）

```
## [heartbeat] 焦点 f12：cc-todo 扫描

**cc-todo 就绪**：
- [{id}] {任务名} | ready 原因：{简述}
（无就绪任务则写"无"）

**aster-todo 待提醒**：
- [{id}] {事项} | 触发原因：{简述}
（无待提醒则写"无"）

建议：{下一步}

**已更新**：heartbeat-pool.md [f12] 上次 → {当前时间}
```

### 其他焦点

```
## [heartbeat] 焦点思考：{焦点名}（{ID}）

**思考结果**：
{具体洞察/建议，2-5 句话}

**建议行动**：
- {下一步，如：加一条 cc-todo / 飞书通知 aster / 记入 MEMORY}

**已更新**：heartbeat-pool.md [{ID}] 上次 → {当前时间}
```

## 资源预算

- 执行时间：≤5 分钟
- 文件读取：heartbeat-pool.md + cc-todo.md + 焦点相关文件（≤5 个）
- 产出长度：回报 ≤20 行

## 边界说明

- **焦点思考超时**：宁可浅想快回，不要深想超时
- **不确定条件是否满足**：标注 `confidence: low`，交给主 cc 判断
- **一次性焦点完成后**：在回报里提醒主 cc 删除该行
