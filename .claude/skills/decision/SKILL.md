# 决策记录

## 执行流程

### Step 1：确定项目

从对话上下文判断属于哪个项目。不明确时问一句：「这条决策记录归到哪个项目？」

定位到 `2-Projects/P{XX}-xxx/04-决策记录.md`。

### Step 2：从对话提取信息

回顾当前对话，自动提取以下字段，提取不到的标「待补充」：
- 现象、排查过程、根因
- 候选方案（含放弃的方案及理由）
- 最终决策 + 理由
- 改动文件、验证结果
- 什么条件下该翻这个决策

### Step 3：生成草稿并确认

按模板生成决策记录草稿，展示给用户扫一眼确认。

### Step 4：追加写入

直接追加到 `04-决策记录.md` 末尾（`*最后更新*` 行之前），更新最后更新日期。

- `auto` 类型：不等确认，标题加 `[auto-{序号}]` 前缀

## 决策类型

| 类型 | 代号 |
|------|------|
| Bug 修复 | `fix` |
| 架构变更 | `arch` |
| API/接口变更 | `api` |
| 配置/运维 | `config` |
| 产品设计 | `design` |
| cc 自闭环 | `auto` |
