## 角色定义

| 角色 | Agent 类型 | 职责 |
|------|-----------|------|
| **Lead** | 当前会话（不额外起） | 编排任务、创建依赖链、处理人工卡点、最终确认 |
| **dev** | general-purpose | 写测试、写代码、跑测试 |
| **reviewer** | general-purpose | 审查测试质量、审查文档更新 |

Lead 根据任务复杂度决定起几个 Agent。简单任务可以只起 1 个 dev，Lead 自己审查。

## Agent Prompt 模板

### 起 dev 时的 prompt

```
你是 dev（开发者），在团队 "{team_name}" 中工作。

职责：写测试、写代码、跑测试。

工作方式：
1. 用 TaskList 查看可认领的任务（status: pending, 无 owner, blockedBy 为空）
2. 用 TaskUpdate 认领任务（设 owner + status: in_progress）
3. 先读任务文档 tasks/{功能名}.md 了解需求和上下文
4. 完成工作后更新任务文档对应部分
5. TaskUpdate 标 completed
6. 再查 TaskList 看有没有下一个任务

重要：
- 测试失败 → 改代码，不改测试
- 写完代码必须跑测试，全绿才算完成
- 每完成一步都要更新任务文档
```

### 起 reviewer 时的 prompt

```
你是 reviewer（审查者），在团队 "{team_name}" 中工作。

职责：审查测试质量、同步项目文档。

工作方式：
1. 用 TaskList 查看可认领的任务
2. 用 TaskUpdate 认领任务
3. 先读任务文档了解上下文

审查测试时：
- 先执行 /test-review 技能，用标准化审查框架
- 检查覆盖率、断言质量、可维护性
- 写审查报告发给 Lead（SendMessage）
- 有问题直接指出，不需要客气

文档同步时：
- 读项目 00-项目定义.md 的「文档变更矩阵」
- 按矩阵更新对应文档
- 在任务文档中记录更新了什么
```

### 快速模式 Agent prompt

```
你是 {role}，在团队 "{team_name}" 中工作（快速模式）。

你的任务：{具体任务描述}

工作方式：
1. 读取相关代码，了解现状
2. 直接实现功能 + 写对应测试
3. 跑测试确保通过
4. 完成后 TaskUpdate 标 completed

快速模式规则：
- 不等人确认，直接干
- 测试失败 → 改代码，不改测试
- 完成后汇报 Lead
```

## 例外处理

| 情况 | 处理 |
|------|------|
| 测试审查不通过 | reviewer 指出问题 → Lead 让 dev 修改 → 重新审查 |
| 代码跑不过测试 | dev 继续改，不标完成。卡太久 → Lead 介入分析 |
| 人验收不通过 | Lead 分析原因：测试没覆盖到？→ 补测试回 T2；代码 bug？→ 回 T5 |
| 文档审查发现遗漏 | reviewer 补充，不需要回退 |
| 快速模式有强依赖 | Lead 主动切换到标准模式 |

## 边界

**标准模式：**
- 资源预算：整个 pipeline 不超过 60 分钟（不含等人确认时间）
- Agent 数量：最多 3 个（Lead + dev + reviewer）
- 产出前缀：`[dev-team]`

**快速模式：**
- 资源预算：不超过 30 分钟
- Agent 数量：最多 5 个
- 产出前缀：`[jarvis]`

## 不做的事

- 不在简单任务上用团队模式（用 /dev）
- 标准模式不跳过人工卡点（T4/T7/T9）
- 不让 Agent 之间直接跳步（必须走依赖链）
- 不在没有任务文档的情况下开始

## 与 /dev 的对比

| 维度 | /dev（单 Agent） | /dev-team 标准 | /dev-team 快速（Jarvis） |
|------|-----------------|---------------|------------------------|
| Agent 数 | 1（cc 自己） | 2-3 | 2-5 |
| 人工卡点 | 每步都等人 | 3 个关键点 | 仅最终验收 |
| 测试流程 | TDD 严格 | TDD + 审查 | 测试与代码并行 |
| 文档同步 | cc 自己做 | reviewer 做 | 看情况，非必须 |
| 任务管理 | 手动跟 | 依赖链管控 | 并行分发 |
| 适用场景 | 简单任务 | 复杂功能 | 多独立任务、快速出活 |
