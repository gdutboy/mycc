# startup-259 参考文档

## 核心理念

> **创业 ≠ 做产品**
> **创业 = 验证关键假设**

大部分创业失败不是执行力问题，是验证顺序错了。

---

## 2-5-9 框架详解

### 2：两类假设

| 类型 | 问题 | 验证阶段 |
|------|------|----------|
| **价值假设** | 有人愿意买单吗？ | 0→1 |
| **增长假设** | 能规模化复制吗？ | 1→100 |

**铁律**：价值假设没过，不谈增长假设。

### 5：五步里程碑

```
需求 → 解决方案 → 商业模式 → 增长 → 壁垒
```

| 步骤 | 核心问题 | 属于 |
|------|----------|------|
| 需求 | 这是真痛点吗？谁在痛？ | 价值假设 |
| 解决方案 | 我的方案真能止痛？ | 价值假设 |
| 商业模式 | 收入 > 成本？可持续？ | 价值假设 |
| 增长 | 怎么快速复制？ | 增长假设 |
| 壁垒 | 凭什么守得住？ | 增长假设 |

### 9：九个要素

| 五步 | 细分要素 |
|------|----------|
| 需求 | ① 细分客户 ② 具体痛点 ③ 核心卖点 |
| 解决方案 | ④ 替代方案（用户现在怎么解决的？） |
| 商业模式 | ⑤ 收入来源 ⑥ 成本结构 ⑦ 核心指标 |
| 增长 | ⑧ 获客渠道 |
| 壁垒 | ⑨ 护城河 |

## 验证方法选择

| 方法 | 适用场景 | 成本 |
|------|----------|------|
| 聊天 | 了解用户痛点 | 极低 |
| 假装客户 | 摸竞品底牌 | 低 |
| 假装面试 | 了解行业数据 | 低 |
| 线下蹲点 | 验证流量/转化 | 中 |
| 落地页测试 | 验证付费意愿 | 中 |
| MVP | 验证完整闭环 | 高 |

## 常见陷阱

### 1. 解决方案先行
- 错：「我要做一个 App...」
- 对：「我发现 XX 人群有 XX 痛点...」

### 2. 伪需求
- 错：「用户说他们需要...」
- 对：「用户现在花钱/花时间在...」（看行为不看嘴）

### 3. 过早谈增长
- 错：「我要投放抖音...」
- 对：「先验证 10 个人愿意付费」

### 4. 验证成本太高
- 错：「我先做 3 个月产品...」
- 对：「我先用 3 天做个落地页测试」

## 从"钱"的视角

| 目标 | 对应模块 | 核心问题 |
|------|----------|----------|
| 能赚钱 | 需求+方案+模式 | 单位经济模型跑通吗？ |
| 赚更多 | 增长 | 能低成本规模化吗？ |
| 赚更久 | 壁垒 | 凭什么别人抄不走？ |

## 阶段判断

| 你在哪 | 该做什么 | 不该做什么 |
|--------|----------|------------|
| 0→1 | 找 PMF，验证价值假设 | 谈融资、招大团队 |
| 1→10 | 优化单元模型，准备复制 | 盲目扩张 |
| 10→100 | 规模化增长，建壁垒 | 改核心产品 |
