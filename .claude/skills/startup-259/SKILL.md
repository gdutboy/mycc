# 创业 259 验证法

## 边界

- 时间预算：≤15 分钟；单次分析一个创业想法
- 产出长度：≤80 行
- 不做：执行验证方案 / 增长设计 / 技术实现 / 编造市场数据

## 执行流程

### Phase 1：听想法
让用户用 1-3 句话描述想法，不打断。

### Phase 2：拆假设（加法）
用 9 要素逐一追问，填满假设拆解表（价值假设 7 项 + 增长假设 2 项）。
要素：细分客户 / 具体痛点 / 核心卖点 / 替代方案 / 收入来源 / 成本结构 / 核心指标 / 获客渠道 / 护城河

### Phase 3：挑关键（减法）
从假设表中挑出 ≤3 个关键假设，三原则：
1. 前置优先（需求没验证，别急着想增长）
2. 单一验证（一次只验一个）
3. 高风险优先（最可能导致失败的先验）

### Phase 4：设计验证
为每个关键假设设计低成本验证方案，包含：验证方法 / 成功标准 / 预计投入 / 下一步。
方法选项：聊天 → 假装客户 → 假装面试 → 线下蹲点 → 落地页测试 → MVP（成本递增）

### Phase 5：输出行动清单
输出带优先级和 Deadline 的行动表（P0/P1/P2）。

## 输出物

1. 假设拆解表（9 要素，标注确定性：猜测 / 有依据）
2. 关键假设（≤3 个，含为什么关键）
3. 验证方案（每个假设一份）
4. 行动清单（带 deadline）

所有产出以 `[startup-259]` 前缀标注。
