### Day 6 - 角色扮演

**深度内容：角色设定的艺术**

角色设定让 CC 成为特定领域的专家。

**基础用法**：
```
你是一个资深前端工程师，帮我审查这段 React 代码
```

**进阶技巧**：

1. **具体化角色**
   ```
   你是一个有 10 年经验的前端架构师，专精 React 性能优化。
   你习惯用 useMemo 和 useCallback 优化组件，
   关注 Core Web Vitals 指标。
   ```

2. **继承上下文**
   ```
   继续用刚才的架构师视角，审查这个新组件
   ```

3. **多角色切换**
   ```
   先以测试工程师视角审查，再以安全专家视角审查
   ```

**练习任务**：为你的项目创建一个"前端架构师"角色，保存到记忆系统

### Day 7 - 分步指令

**深度内容：复杂任务的分解与控制**

分步指令让 CC 按你规划的路径工作。

**基础用法**：
```
先分析这个需求，然后写一个实现计划，批准后再执行
```

**进阶技巧**：

1. **用 TodoWrite 跟踪**
   ```
   创建一个任务列表：
   1. 分析现有 API 结构
   2. 设计新接口
   3. 实现业务逻辑
   4. 写单元测试
   ```

2. **设置检查点**
   ```
   完成第一步后停下来，让我确认方向正确再继续
   ```

3. **条件分支**
   ```
   如果发现数据库需要迁移，告诉我；如果不需要，直接开始实现
   ```

**练习任务**：用分步方式实现一个小功能，每步都让 CC 停下来确认

### Day 8 - 参考风格

**深度内容：Few-shot 学习与风格控制**

通过示例让 CC 精确理解你的期望。

**基础用法**：
```
用这种风格重写：
const add = (a, b) => a + b;
```

**进阶技巧**：

1. **多示例注入**
   ```
   用以下风格写代码：

   // 示例1
   const getUser = async (id) => {
     const res = await fetch(`/api/users/${id}`);
     return res.json();
   };

   // 示例2
   const getPosts = async (userId) => {
     const res = await fetch(`/api/users/${userId}/posts`);
     return res.json();
   };

   // 现在写 getComments
   ```

2. **输出格式示例**
   ```
   用以下 JSON 格式输出：
   {
     "file": "路径",
     "issues": ["问题1", "问题2"],
     "severity": "high|medium|low"
   }
   ```

3. **风格参考文档**
   ```
   参考这个文档的代码风格：https://xxx/style-guide.md
   ```

**练习任务**：让 CC 用指定格式输出代码审查报告

### Day 9 - 约束条件

**深度内容：精确表达边界与限制**

约束让 CC 知道什么是"不可以"的。

**基础用法**：
```
不要用第三方库，代码不超过 100 行
```

**进阶技巧**：

1. **否定指令要具体**
   ```
   ❌ 不要用任何库
   ✅ 不要用 lodash 或其他工具库，只用原生 JS
   ```

2. **多重约束**
   ```
   实现这个功能，满足以下约束：
   - 使用 TypeScript
   - 不能用 any 类型
   - 保持函数单一职责
   - 导出类型要完整
   ```

3. **边界条件**
   ```
   处理以下边界情况：
   - 空数组
   - null/undefined
   - 超长输入（>10000字符）
   ```

**练习任务**：让 CC 生成符合严格约束的代码

---

### Day 10 - 迭代优化

**深度内容：反馈循环与持续改进**

迭代让结果逐步逼近完美。

**基础用法**：
```
再优化一下性能
```

**进阶技巧**：

1. **具体迭代目标**
   ```
   第一轮：让代码能跑起来
   第二轮：优化到 100ms 内
   第三轮：减少内存占用
   ```

2. **保留上下文**
   ```
   继续刚才的优化，这次关注可读性
   ```

3. **对比改进**
   ```
   给出三个版本：原始版、优化版、最佳版，解释每个的取舍
   ```

**练习任务**：用 3 轮迭代优化一个函数，对比每轮的结果

---

### Day 11 - 组合技：工具链编排

**深度内容：从探索到提交的全流程**

整合所有工具完成一个完整任务。

**工作流模式**：
```
1. 探索：Glob 找文件 → Read 读代码 → Grep 搜用法
2. 分析：用 Agent 做深入分析
3. 实现：Edit/Write 修改代码
4. 验证：Bash 运行测试
5. 提交：git add + commit
```

**练习任务**：完成一个小功能，从探索到提交
