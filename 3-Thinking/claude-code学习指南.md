# Claude Code 5分钟学习指南

> 每天5分钟，30天成为 CC 高手

---

## 第1周：基础入门

### Day 1 - 基础对话
- **内容**：直接用自然语言下达任务
- **示例**：`帮我写一个计算器`
- **技巧**：描述你要什么，而不是怎么写

### Day 2 - 文件操作
- **内容**：`Read` 读文件、`Edit` 改文件、`Write` 写新文件
- **示例**：读取代码后让 CC 解释
- **技巧**：CC 会记住对话上下文，不用重复说明

### Day 3 - 代码审查
- **内容**：让 CC 审查代码、找 bug
- **示例**：`检查这个文件有什么问题`
- **技巧**：可以指定审查标准（性能、安全、可读性）

### Day 4 - 项目探索
- **内容**：让 CC 了解整个项目
- **示例**：`帮我看看这个项目是做什么的`
- **技巧**：用 `Glob` 找文件，`Grep` 搜代码

### Day 5 - 任务执行
- **内容**：用 `Bash` 执行命令行
- **示例**：`运行 npm install`
- **技巧**：危险命令会提示确认

## 第2周：提示词技巧

### Day 6 - 角色扮演
- **内容**：给 CC 设定角色
- **示例**：`你是一个资深前端工程师，帮我审查代码`
- **技巧**：角色设定越具体，回复越专业

### Day 7 - 分步指令
- **内容**：复杂任务分步骤下达
- **示例**：先让 CC 规划，再让执行
- **技巧**：用 `TodoWrite` 跟踪进度

### Day 8 - 参考风格
- **内容**：让 CC 模仿某种风格
- **示例**：`用 React Hooks 风格重写`
- **技巧**：可以指定参考代码或文档

### Day 9 - 限制条件
- **内容**：明确告诉 CC 约束条件
- **示例**：`不要用第三方库`、`代码不超过 100 行`
- **技巧**：约束要具体明确

### Day 10 - 迭代优化
- **内容**：让 CC 不断改进
- **示例**：`再优化一下性能`、`换个写法`
- **技巧**：CC 会记住之前的修改
