## 问题
Windows 环境下 `claude -p` 无法正确接收多行 prompt，CC 收到的是被截断的消息。

## 已尝试的方案
1. `-p` 直接传多行 → ❌ 截断
2. `-f` 文件方式 → ❌ 用户 CLI 不支持
3. 单行 `|` 分隔 → ❌ 超时
4. 多行 prompt → ❌ 截断
5. PowerShell Get-Content 管道 → ❌ 无输出
6. **stdin 写入 + end()** → ✅ 成功！

## 应用场景

### 适合
- **技术选型/方案对比**：多角度分析，CC 汇总辅助决策
- **复杂开放问题**：架构设计、算法方案，不同模型思路互补
- **知识盲区探索**：交叉验证降低单模型幻觉风险
- **头脑风暴**：产品创意、文案，风格各异

### 不适合
- 简单事实查询（一个引擎够了）
- 写具体代码（三份代码难合并）
- 紧急秒回场景（总耗时 1-2 分钟）

### 注意
- 三个引擎都收费，不是免费的，按需使用

### 延伸方向
- 自动路由：判断问题复杂度，简单走单引擎，复杂升级团队模式
- 投票机制：答案相似度高 → 高置信；差异大 → 标记需人工判断
