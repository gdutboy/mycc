## Day 5 - 任务执行：用 Bash 让 CC 帮你跑命令

**今日主题**：把 CC 变成你的终端小助手

### 为什么用 Bash？

CC 最强大的能力不只是读代码、写代码，而是**直接帮你执行**。安装依赖、运行脚本、Git 操作，这些重复性的敲命令体力活，都可以交给 CC。

### 核心用法

| 场景 | 怎么说 | CC 做什么 |
|------|--------|-----------|
| 安装依赖 | `帮我运行 npm install` | 执行并显示输出 |
| 启动服务 | `启动 npm run dev` | 后台运行并监控 |
| Git 操作 | `提交代码` | 自动 add + commit + 提示 push |
| 查看状态 | `git status` | 显示当前仓库状态 |

### 危险提示

以下命令 CC 会**请求确认**后才执行：
- `rm -rf` 删除文件
- `git push --force` 强制推送
- `npm uninstall` 卸载依赖
- 任何涉及生产环境的操作

### 今日练习（必做！）

1. 对 CC 说：`帮我运行 npm run dev`，观察输出
2. 对 CC 说：`git status`，看看当前项目状态
3. 尝试：`帮我执行 npm install && npm run build`

### 进阶技巧

- **看输出**：CC 会显示完整命令输出，方便排查问题
- **持续监控**：可以说`帮我监控这个服务，有异常通知我`
- **组合命令**：可以说`先安装依赖，再运行测试，最后打包`
