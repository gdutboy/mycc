# Day 5 - 任务执行：用 Bash 让 CC 帮你跑命令

> 目标：把 CC 变成你的终端执行力

---

## 为什么用 Bash？

CC 最强大的能力不只是读代码、写代码，而是**直接帮你执行**。

安装依赖、运行脚本、Git 操作——这些重复性的敲命令体力活，都可以交给 CC。

---

## 核心用法

### 安装依赖

```
帮我运行 npm install
```

CC 会执行并显示完整输出，包括：
- 安装进度
- 依赖树
- 警告信息（如有）

### 启动服务

```
启动 npm run dev
```

CC 会：
- 执行命令
- 监控输出
- 遇到错误会分析原因

### Git 操作

```
帮我提交代码
```

CC 会：
- 自动 add 所有文件
- 生成符合规范的 commit message
- 提示你是否要 push

### 组合命令

```
先安装依赖，再运行测试，最后打包
```

CC 会按顺序执行，并汇总结果。

---

## 危险命令确认

以下命令 CC 会**请求确认**后才执行：

| 命令类型 | 示例 | 风险 |
|---------|------|------|
| 删除文件 | `rm -rf` | 误删重要文件 |
| 强制推送 | `git push --force` | 覆盖远程代码 |
| 卸载依赖 | `npm uninstall` | 破坏环境 |
| 数据操作 | `DROP TABLE`、`DELETE FROM` | 数据丢失 |
| 生产环境 | 任何带 production/prod 的 | 业务中断 |

---

## 管道组合（高级技能）

CC 支持管道，可以做复杂数据处理：

```bash
# 分析错误日志
cat error.log | claude -p "分析这些错误，找出最常见的 3 个问题"

# 检查代码问题
git diff | claude -p "审查这些改动有什么问题"

# 批量重命名
ls *.txt | claude -p "把这些文件重命名为 .md 后缀"
```

## 进阶技巧

### 1. 持续监控

```
帮我监控 npm run dev，有异常通知我
```

CC 会：
- 启动服务
- 实时监控输出
- 发现错误立即报告

### 2. 后台运行

```
在后台启动 npm run dev，然后继续做别的事
```

### 3. 脚本化

```
写一个自动测试脚本：运行测试，生成覆盖率报告，然后如果失败则通知我
```
